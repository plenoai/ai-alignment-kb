# Symphony — OpenAI 自律コーディングオーケストレータ

OpenAI が開発した、タスクトラッカー（Linear）の課題を自律エージェントに分配し、隔離されたワークスペースで並列実行するオーケストレーションシステム。「エージェントを監督する」から「作業を管理する」への移行を目指す。

## 概要

- **リポジトリ:** https://github.com/openai/symphony
- **実装言語:** Elixir（参照実装）
- **ステータス:** 低調な工学的プレビュー（2025年時点）
- **前提:** ハーネスエンジニアリングを適用済みのコードベースでの使用を推奨

## アーキテクチャ

Symphony のオーケストレータは5状態のステートマシンで課題のライフサイクルを管理する：

```
Unclaimed ──→ Claimed ──→ Running ──→ Released
                              │
                              └──→ RetryQueued ──→ Claimed
```

| 状態 | 意味 |
|------|------|
| Unclaimed | 未割り当て。ポーリングで検知待ち |
| Claimed | リソース確保済み。ワークスペース準備中 |
| Running | エージェントが課題に取り組み中 |
| RetryQueued | 失敗後、バックオフ付きで再試行待機 |
| Released | リソース解放完了 |

### ワークスペース隔離

Symphony の最重要安全不変条件：

1. エージェントは課題ごとの `workspace_path` 内でのみ実行される
2. ワークスペースパスは `workspace_root` 配下であることがプリフィックス検証される
3. ディレクトリ名は `[A-Za-z0-9._-]` に正規化される（インジェクション防止）

### ガードレール

| 制約 | デフォルト値 | 目的 |
|------|------------|------|
| `max_concurrent_agents` | 10 | グローバルスロット上限 |
| `agent.max_turns` | 20 | 1課題あたりのターン数上限 |
| `codex.stall_timeout_ms` | 300,000 (5分) | 無応答ワーカーの強制終了 |
| `agent.max_retry_backoff_ms` | 300,000 (5分) | 再試行バックオフ上限 |

### 承認ポリシー

Symphony は単一の承認方式を強制せず、実装定義としている：

- **高信頼環境:** 自動承認（CI + PR レビューが事後ゲート）
- **制限環境:** オペレータ承認必須
- **混合:** サンドボックスポリシーの組み合わせ

ユーザー入力が必要になった場合は「ハード失敗」として実行を停止し、再試行をスケジュールする。

## 観察されたパターン

### [Read-Write 境界](./read-write-boundary.md)

エージェントはワークスペース内で自律的にコードを書くが（Read フェーズの拡張）、本番への統合は PR マージという人間承認ゲートを経由する。承認ポリシーを実装定義にすることで、環境の信頼度に応じて境界の位置を調整できる。

### [階層的エージェント監視](./hierarchical-agent-oversight.md)

オーケストレータがメタエージェントとして機能し、複数のコーディングエージェントをグローバルスロット・ストール検知・リトライ管理で統括する。個別エージェントの行動は監視せず、状態遷移と全体指標で管理する。

### [クローズドループ自律改善](./closed-loop-autonomous-improvement.md)

ターンキャップ（デフォルト20）とストールタイムアウトが終了条件として機能し、RetryQueued → Claimed の再試行ループが自動安全網を構成する。CI ステータスが品質ゲートとなる。

### [ハーネスエンジニアリング](./harness-engineering.md)

Symphony の仕様書は「ハーネスハードニング」を明示的に定義し、リポジトリ・トラッカー・プロンプトを「完全に信頼できない」と仮定する。ワークスペースパスのプリフィックス検証、Linear GraphQL のスコープ制限、発火対象課題の事前フィルタリングがこの概念の具体的実装。

### [エージェント自律性のスペクトラム](./agent-autonomy-spectrum.md)

Symphony は L3〜L4（監視あり〜例外時のみ）で動作する。エージェントは自律的にコードを書き、オーケストレータが例外（ストール、リトライ上限超過）を検知して介入する。人間の関与は PR レビュー時に集中する。
