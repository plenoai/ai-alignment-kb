# 階層的エージェント監視

多数の自律エージェントを直接監視するのではなく、上位 AI（メタエージェント）がグローバル状態を集約して監視し、必要な場合のみ介入する。エージェント数が増えても監視コストが増大しない。

## コンテキスト

多数の自律エージェントが並列に動作している。各エージェントは局所的には最適な行動をとるが、全体としての品質・目標達成・体験の一貫性は誰も保証していない。エージェントの一体一体を人間が監視するコストは、エージェント数に比例して増大する。

## 問題

**自律エージェントを増やすほどシステムの能力は上がるが、全体品質を保証するための監視コストも線形に増大する。エージェント数のスケールに監視がついてこられない。**

## フォース

- 全エージェントを個別に監視するとコストが O(n) で増大する
- 各エージェントは局所最適を追求するが、全体整合性は自然には生まれない
- 介入しすぎると自律エージェントのメリット（並列性・専門性）が失われる
- 介入しなさすぎると全体が目的から逸脱する
- エージェント間の相互作用は創発的で、個別観察では予測できない

## 解決策

**上位のメタエージェントを置き、個々のエージェントの行動ではなく「全体の状態」を監視させる。メタエージェントは状態集約を受け取り、全体目標に対する逸脱を検知したときだけ下位エージェントに介入・調整する。**

```
[メタエージェント]
  ├── グローバル状態を監視（個別行動ではなく全体指標）
  ├── 目標からの逸脱を検知
  └── 必要時のみ介入（全体の流れを調整）
         │
    ┌────┴────────────────┐
    ▼                    ▼
[エージェント A]    [エージェント B]  ···  [エージェント N]
  自律的に動作        自律的に動作           自律的に動作
  局所最適を追求      局所最適を追求         局所最適を追求
         │                 │
    状態を報告         状態を報告
```

### 監視設計の原則

| 監視対象 | 粒度 | 理由 |
|---------|------|------|
| 個別エージェントの行動 | 見ない（原則） | O(n) コストを避ける |
| 全体指標・集約状態 | 常時監視 | 全体整合性の代理指標 |
| 逸脱・異常 | 閾値超過時のみ | 例外ベースの介入で効率化 |

### メタエージェントの責務

1. **状態集約:** 下位エージェント群の状態を集約して全体像を把握
2. **目標整合:** 全体目標に対する達成度を評価
3. **選択的介入:** 逸脱を検知したときだけ下位エージェントの行動を変える
4. **ジレンマ生成（応用）:** 意図的に緊張状態を作り、システム全体の活性を調整

## 結果コンテキスト

**得られるもの**
- 監視コストが O(n) → O(1) に近づく（全体指標のみを監視）
- エージェント数を増やしても監視の質が劣化しない
- 各エージェントの自律性を尊重しつつ全体整合性を確保
- 局所的な混乱が全体に伝播するのを防げる

**失うもの**
- 個別エージェントの行動の可視性が低下する
- メタエージェントが全体指標の設計を誤ると、問題を見逃す
- メタエージェント自体が単一障害点になる
- 全体指標では捉えられない「静かな劣化」が蓄積するリスク

## 既知の使用例

- **ゲームAI / メタAI（三宅陽一郎）:** メタAIが多数のキャラクターAIを俯瞰し、プレイヤーの感情・スキルに応じてゲーム全体を動的調整。エージェント数が増えても制御品質を維持。 → [マルチエージェントシステム](./multi-agent-system.md)
- **スマートシティAI:** 都市規模のマルチエージェントに同アーキテクチャを適用。メタAIが都市サービスを制御し、個別エージェントの満足度を集約して政策を調整。
- **クローズドループ自律改善（Ralph Loop）:** ラウンド終了後の振り返りフェーズが、改善サイクル全体を俯瞰するメタレイヤーとして機能。 → [クローズドループ自律改善](./closed-loop-autonomous-improvement.md)
- **マイクロサービス監視:** サービスメッシュ（Istio 等）が個別サービスの健全性を集約し、トラフィック制御で全体を調整。
- **Cursor クラウドエージェント:** 開発者がメタレベルで複数の自律エージェントを監視。個別のコード行ではなく成果物（ログ・動画・プレビュー）ベースで介入判断する。社内 PR の 1/3 以上がエージェント起源。 → [Cursor 事例](./cursor-third-era-case.md)
- **Symphony（OpenAI）:** オーケストレータがメタエージェントとして複数コーディングエージェントをグローバルスロット・ストール検知・リトライ管理で統括。個別行動は監視せず状態遷移と全体指標で管理。 → [Symphony 事例](./symphony-case.md)
- **OpenAI — AI Discovered Unknowns:** AI ジャッジがメタエージェントとして ChatGPT の全会話を俯瞰的に監視し、個別会話ではなく全体パターンでミスアライメントを検出。 → [OpenAI AI Discovered Unknowns 事例](./openai-ai-discovered-unknowns-case.md)
- **OpenAI Agentic Governance（2023）:** プラクティス 5「自動モニタリング」で第二の監視 AI がプライマリエージェントの推論・アクションを審査する構造を提案。再帰的脆弱性（プロンプトインジェクションが監視 AI にも波及）とコスト課題を明示。 → [OpenAI エージェント型 AI ガバナンス事例](./openai-agentic-governance-case.md)
- **OpenAI — コード検証スケーリング:** 自動コードレビューシステムがメタレビュアーとして月間 100 万件以上の PR を監視。重要な問題に焦点を当て、人間の判断を支援。 → [OpenAI コード検証事例](./openai-scaling-code-verification-case.md)
