# Claude Ads: マルチプラットフォーム広告監査スキル

**Source:** https://github.com/AgriciDaniel/claude-ads
**Date Added:** 2026-03-03

## 概要

Claude Code 向けの包括的な有料広告監査・最適化スキル。主要広告プラットフォームを横断した体系的な分析と自動化された推奨事項を提供する。

## 対応プラットフォームとチェック数

| プラットフォーム | チェック数 |
|-----------------|-----------|
| Google Ads | 74 |
| Meta/Facebook | 46 |
| LinkedIn Ads | 25 |
| TikTok Ads | 25 |
| Microsoft/Bing Ads | 20 |
| **合計** | **190** |

## アーキテクチャ設計

### モジュラー設計（スケーラブルオーバーサイトの観点）

```
中央オーケストレーター
├── 専門サブスキル（プラットフォーム別）
├── 参照データファイル（RAG パターン）
└── 業界別テンプレート（11 業種）
```

#### 並列サブエージェント処理
- 6 つの並列監査エージェントが同時にプラットフォームを分析
- 人間が順序立てて行うレビューをマルチスレッド化

### 品質ゲートの強制
ベストプラクティスの品質ゲートを自動適用 → 人間の判断バイアスを排除

## 出力フォーマット

- **Ads Health Score (0-100):** A〜F の文字グレード付き
- 分析軸: コンバージョントラッキング、予算配分、クリエイティブ品質、コンプライアンス、入札戦略

## スケーラブルオーバーサイトへの示唆

### 1. 複数エージェントによる分散監査
単一エージェントではなく、プラットフォーム特化の専門エージェントを並列実行することで：
- 各エージェントの専門性が深まる（汎用 vs 専門のトレードオフ）
- 相互に検証可能な構造（一方のエージェントが他方の死角を補完）

### 2. RAG パターンによる知識のスケール
- 参照データファイルをオンデマンドで読み込む設計
- モデルのパラメータではなく外部ファイルに知識を置くことで更新・拡張が容易

### 3. 業種別テンプレートの示唆
11 業種（SaaS, EC, B2B 等）への特化は：
- 汎用スコアリングの限界を示す（コンテキストが重要）
- ドメイン知識なしの AI 監査は「形式的なチェック」に留まるリスク

### 4. インストール・配布の容易さ
ワンコマンドインストール → スキルの横展開がしやすい設計
Claude Code のスキルシステムそのものがスケーラブルオーバーサイトの一形態
