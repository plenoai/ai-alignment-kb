# Claude Code による Google 広告監査

**Source:** https://stormy.ai/blog/claude-code-google-ads-audit-guide
**Date Added:** 2026-03-03

## 概要

Claude Code（Anthropic の CLI ツール）と MCP サーバーを組み合わせて、Google 広告アカウントを自動監査・最適化する手法。手動管理からエージェント型 AI 駆動最適化への移行を実現する。

## スケーラブルオーバーサイトの観点

### 問題設定

広告管理は専門知識を要する反復作業であり、人間の監視コストが高い領域：
- 不正クリック: 広告クリックの約 5.1% が不正（世界で年間 ~380 億ドルの損失）
- 無駄な支出: 最適化なしでは予算の最大 35% が無駄に
- ブランド検索広告の 20-30% が競合不在でオーガニック流入に課金
- ターゲティング不良・入札戦略の陳腐化による損失が約 25%

### 人間の監視 vs AI 監査のトレードオフ

| 指標 | 手動 | Claude Code |
|------|------|-------------|
| 監査時間 | 4〜8 時間 | 5 分以内 |
| チェック数 | 人的エラーあり | 190+ 自動化 |

### スケーラビリティを実現する設計パターン

#### 1. MCP ブリッジによる API 統合
- Claude Code → MCP サーバー → Google Ads API の接続
- 自然言語指示をアクション可能な API コールに変換
- 人間の判断を介さずにデータ取得・変更を実行

#### 2. "Ralph Wiggum" 自動化ループ（反復最適化の例）
三段階プロセス：
1. 高コスト・ゼロコンバージョンキーワードの特定
2. オークションコストを引き上げるキーワードカニバリゼーションの検出
3. ネガティブキーワードリストを API 経由で自動コミット

**効果:** 午後の作業が 30 秒のタスクに短縮

#### 3. 重点監査チェック（人間がミスしやすい領域）
- **コンバージョントラッキングの二重計測:** レガシータグと新タグの競合
- **Performance Max アセットギャップ分析:** 見出し・画像の不完全によるペナルティ
- **"クリック最大化" 戦略の検出:** 低インテントモバイルトラフィックのトラップ

## スケーラブルオーバーサイトへの示唆

1. **ドメイン知識の構造化がスケールの鍵:** 190 のチェックリストは人間の専門知識をコード化したもの。AI は構造化された知識を拡張する
2. **エージェントの自律性と人間の監視の境界:** 読み取り（監査）は完全自動化、書き込み（最適化実行）は人間確認を挟む設計が安全
3. **定量的な健全性スコア:** 0-100 のスコアリングで人間が俯瞰できる透明性を確保
