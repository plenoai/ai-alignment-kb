# OpenAI — コード検証のスケーリング

OpenAI がリポジトリ全体へのアクセスと実行能力を持つ自動コードレビューシステムを構築し、月間 100 万件以上のプルリクエストを処理する事例。「安全性は採用されなければ機能しない」という実装的洞察に基づく。

## 概要

- **発表:** 2025年12月
- **組織:** OpenAI Alignment チーム
- **規模:** 月間 100 万件以上の PR を処理（2025年10月時点）
- **アクション率:** コメント付き PR の 52.7% で著者が修正を実施
- **肯定的反応:** 外部評価反応の 80% 以上がポジティブ

## 設計原則

### リポジトリ全体アクセス

従来の差分情報のみの提供から、全コードベースの文脈を利用することで精度向上。

### 信号対雑音比の優先

「再現率と精度のトレードオフを意識的に受け入れ、信号対雑音比を優先」する設計。ユーザーの採用を阻害しない「低い安全コスト」を目指す。

### 訓練時と展開時の分離

| フェーズ | 特性 | 理由 |
|---------|------|------|
| 訓練用検証器 | 過敏性を容認（誤検知多数でも許容） | 安全側に倒す |
| 展開用検証器 | 実世界のあいまいさに対応 | ユーザー信頼の構築 |

## 結果

- 全社内 PR が自動審査対象
- GPT-5.1-Codex は GPT-5 比で再現率と精度の両方を改善
- 本番環境で重大な欠陥検出と本番ブロック問題の防止を実現
- 検証能力がコード生成能力と並行して進化

## 観察されたパターン

### [検証天井](./verification-ceiling.md)

「モデルが強力なジェネレータになるにつれ、その検証・批判・判断支援能力も拡張されるべき」という主張は検証天井パターンの直接的示唆。検証能力の向上が自律性の天井を引き上げる。

### [階層的エージェント監視](./hierarchical-agent-oversight.md)

自動コードレビューシステムがメタレビュアーとして機能し、月間 100 万件以上の PR を個別にレビューせず重要な問題に焦点を当てる。人間の判断を支援するツールとして位置づけ。

### [ドメイン知識コード化](./domain-knowledge-as-code.md)

リポジトリ全体のコンテキストを利用したレビューは、コードベースに蓄積されたドメイン知識をレビュー判断に活用する。
