# メタ改善ループ

AI エージェントが対象システムを改善するだけでなく、各サイクルの結果を振り返り、自身の改善手法そのものを更新する。改善の質がラウンドを重ねるごとに向上する。

## コンテキスト

AI エージェントが自律的に改善サイクルを反復している。各ラウンドで得られた結果（成功した修正・失敗したリバート・残った Issue など）は、次のラウンドをより賢くするための情報として利用可能な状態にある。

## 問題

**同じ改善手法を何ラウンド繰り返しても、手法自体が固定されていれば改善の質は上がらない。初期ラウンドで効果が低かったアプローチを延々と繰り返すだけになる。**

## フォース

- ラウンドごとの結果は改善手法の有効性を示す貴重なシグナルだが、捨てられることが多い
- 手法を毎ラウンド変えすぎると収束せず、比較評価もできなくなる
- 振り返りに時間をかけすぎると改善サイクル自体が遅くなる
- 自己改訂が暴走すると、元の目的から逸脱した手法に変化するリスクがある
- 人間がメタレベルの変化を把握していないと、エージェントの行動が予測不能になる

## 解決策

**改善サイクルの外側にメタループを設ける。各ラウンド終了後に結果を記録（振り返り）し、その記録をもとに次ラウンドで使う改善手法を更新（自己学習）する。オブジェクトレベルの改善とメタレベルの学習を分離する。**

```
[オブジェクトレベル: 対象システムの改善]
  Round N: QA → Fix → Refactor
           ↓
        結果記録
           ↓
[メタレベル: 改善手法の改善]
  振り返り（何が効いて何が効かなかったか）
           ↓
  自己学習（次ラウンドの改善手法を更新）
           ↓
  Round N+1: 更新された手法で実行
```

### 振り返りで記録すべき情報

| 情報 | 活用先 |
|------|--------|
| リバートされた修正のパターン | 避けるべき変更の種類を学習 |
| クローズできなかった Issue の傾向 | 苦手な問題領域を特定 |
| テストを通過した修正のアプローチ | 有効な手法として強化 |
| ラウンドあたりの Issue 削減数 | 手法の有効性の定量評価 |

### メタ改訂の安全境界

自己改訂が目的から逸脱しないよう、変更できるスコープを制限する：

- **変更可能:** 問題アプローチの優先順位、使用するツールの順序、チェックの粒度
- **変更不可:** 終了条件、安全網（E2E テスト・自動リバート）、人間への報告形式

## 結果コンテキスト

**得られるもの**
- 改善の質がラウンドを重ねるごとに向上する（学習曲線の実現）
- 同じ失敗を繰り返さない（失敗からの学習）
- 長期実行ほど効率が上がる（初期コストの回収）

**失うもの**
- 手法が変化するため、ラウンド間の比較が難しくなる
- メタレベルの変化を人間が追跡しないと、エージェントの意思決定が不透明になる
- 振り返り・自己学習のフェーズがラウンドの実行時間を延ばす
- 自己改訂の暴走リスク（安全境界の設計が不十分な場合）

## 既知の使用例

- **Ralph Loop:** 各ラウンド後に振り返りフェーズで結果を記録し、自己学習フェーズで改善手法を更新。対象コードと手法の両方が改善される。 → [Ralph Loop 事例](./ralph-loop-case.md)
- **強化学習エージェント:** 報酬シグナルをもとにポリシー自体を更新する構造はメタ改善ループの形式化
- **AlphaGo / AlphaZero:** 自己対戦の結果で評価関数と戦略を同時に改善
