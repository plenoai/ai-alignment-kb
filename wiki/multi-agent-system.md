# メタAI

多数の自律エージェントが個別に動作する環境で、全体を俯瞰する上位 AI がグローバル状態を監視し、ゲーム（システム）全体の流れを動的に制御する。三宅陽一郎（スクウェア・エニックス）が体系化したゲーム AI アーキテクチャの中核概念。

## ゲームAIの3層アーキテクチャ

三宅陽一郎氏は、現代のゲームAIを3種のAIが連携する分散協調システムとして定義した（2015年頃に完成、現在は世界標準）。

```
┌─────────────────────────────────────┐
│  メタAI（全体統括）                   │
│  ゲーム全体を俯瞰・ドラマを生成         │
│  プレイヤーの感情・スキルを推定         │
└───────────────┬─────────────────────┘
                │ 指示・調整
    ┌───────────┴──────────┐
    ▼                      ▼
┌────────────┐    ┌──────────────────┐
│ キャラクターAI │    │ スパーシャルAI      │
│ 個々のNPCの   │    │ 地形・空間を認識し   │
│ 自律的思考    │    │ 他AIに情報を供給    │
└────────────┘    └──────────────────┘
```

| AI 種別 | 役割 | 視野 |
|--------|------|------|
| メタAI | ゲーム全体の流れ・難易度・ドラマを制御 | グローバル |
| キャラクターAI | 個々のNPCが自律的に思考・行動 | ローカル |
| スパーシャルAI | 地形・空間の特徴を把握し情報を供給 | 空間的 |

## メタAIが解く問題

マルチエージェントが各自で勝手に動くと混乱が生じる。個々のキャラクターAIは局所的に最適な行動をとるが、**全体としてのドラマや体験は誰も保証しない**。

メタAIはこの問題を「俯瞰的な制御層」として解決する：
- プレイヤーのスキル・感情・意図をリアルタイムに推定
- ゲームの流れ・難易度を動的に調整（得意な領域ならダンジョン追加、報酬を好むなら出現率を変更）
- マルチエージェントの行動を「一つのドラマ」として統合
- ジレンマ（葛藤）を意図的に生成してプレイヤーの感情を動かす

## スケーラブルオーバーサイトとしての意義

メタAIは**エージェントの数が増えても監視コストが増大しない**仕組みを実現している：

- キャラクターAIは**自律的に動く**（一体一体を人間が制御しない）
- メタAIは**状態の集約**を受け取り、全体指標だけを監視する
- 介入が必要な場合のみメタAIが「流れを変える」

これは人間の監視者がスケールできない問題に対して、AIがAIを監視するという構造そのもの。

## スマートシティへの応用

三宅氏は立教大学で同アーキテクチャをスマートシティ研究に適用：

- メタAI → 都市全体のサービス制御
- キャラクターAI → 個別市民エージェント
- スパーシャルAI → 都市空間の認識

Unity エンジン上でシミュレーションし、マルチエージェントの満足度の変化を解析している。

## 観察されたパターン

- [階層的エージェント監視](./hierarchical-agent-oversight.md) — メタAIはこのパターンの典型実装。上位AIが下位エージェント群を俯瞰し、個別介入なしに全体を制御する
- [並列専門エージェント](./parallel-specialist-agents.md) — キャラクターAI群は並列専門エージェントとして動作する
- [定量ヘルススコア](./quantitative-health-score.md) — プレイヤーの感情・スキル推定値がメタAIの制御入力になる
