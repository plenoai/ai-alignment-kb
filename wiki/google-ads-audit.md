# Claude Code による Google 広告監査

Claude Code と MCP サーバーを組み合わせて Google 広告アカウントを自動監査・最適化した事例。

**Source:** https://stormy.ai/blog/claude-code-google-ads-audit-guide
**Date:** 2026-03-03

## 概要

手動で 4〜8 時間かかる Google 広告監査を、Claude Code で 5 分以内に完了させる手法。

| 指標 | 手動 | Claude Code |
|------|------|-------------|
| 監査時間 | 4〜8 時間 | 5 分以内 |
| チェック数 | 人的エラーあり | 190+ 自動化 |

## 背景にある問題

広告管理は専門知識を要する反復作業で、人間の監視コストが高い：
- 不正クリックが広告クリックの約 5.1%（世界で年間 ~380 億ドルの損失）
- 最適化なしでは予算の最大 35% が無駄に
- ブランド検索広告の 20-30% が競合不在でオーガニック流入に課金

## 反復最適化ループ（"Ralph Wiggum" プロセス）

1. 高コスト・ゼロコンバージョンキーワードの特定
2. キーワードカニバリゼーションの検出
3. ネガティブキーワードリストを API 経由で自動コミット

→ 午後の作業が 30 秒のタスクに短縮

## 観察されたパターン

- [Read-Write 境界](./read-write-boundary.md) — 監査（読み取り）は完全自動、書き込み（最適化実行）は人間確認
- [ドメイン知識コード化](./domain-knowledge-as-code.md) — 190 のチェックリストに専門知識を構造化
- [定量ヘルススコア](./quantitative-health-score.md) — 0-100 のスコアで人間が俯瞰できる透明性を確保
