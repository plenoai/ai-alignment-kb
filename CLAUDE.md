# Scalable Oversight Framework — 編集方針

## リポジトリの目的

AI エージェントに対する人間の監視（Oversight）を、品質を落とさずスケールさせるパターン集。分野横断的に事例を収集し、再利用可能なパターンとして抽象化する。

## ディレクトリ構成と役割

| ディレクトリ | 役割 | 追加基準 |
|------------|------|---------|
| `patterns/` | 再利用可能な設計パターン | 複数の事例で観察された抽象的な型 |
| `wiki/` | 概念・用語の解説 | パターンを理解するための前提知識 |
| `knowledge/` | 事例・ソース素材 | パターン抽出の一次資料 |

## ナレッジ追加のワークフロー

1. `knowledge/` に事例ファイルを追加する
2. 「スケーラブルオーバーサイトへの示唆」セクションでパターンとの接続を記述する
3. 既存パターンに合致する → パターンファイルの「既知の使用例」に追記する
4. 新しいパターンを発見した → `patterns/` に新規ファイルを作成する

## パターンファイルの形式

各パターンは以下のセクションで構成する（Wiki の [Pattern Language](https://en.wikipedia.org/wiki/Pattern_language) 形式に準拠）：

- **意図** — 何の問題を解くか
- **動機** — なぜそのパターンが必要か（背景の問題）
- **構造** — どのように構成されるか（図や表を使う）
- **帰結** — 得られるもの・失うもの（トレードオフを必ず記述）
- **既知の使用例** — 実世界での適用事例
- **関連パターン** — 他パターンへのリンク
- **ソース** — 対応する `knowledge/` ファイルへのリンク

## 分野について

分野（広告、コード、医療など）による分類は行わない。**パターン軸で整理する**ことを優先する。同じパターンが異なる分野に現れることがこのリポジトリの主要な発見になる。
