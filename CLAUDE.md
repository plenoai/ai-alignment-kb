# Scalable Oversight Framework — 編集方針

## リポジトリの目的

AI エージェントに対する人間の監視（Oversight）を、品質を落とさずスケールさせるパターン集。分野横断的に事例を収集し、再利用可能なパターンとして抽象化する。

## 構造

全てのコンテンツは `wiki/` 配下のフラットなWikiページとして管理する。パターン・概念・事例を区別するサブディレクトリは持たない。ページ間のリンクが構造を表現する。

| ページ種別 | 判断基準 | 形式 |
|-----------|---------|------|
| コンセプト | 概念・用語の解説 | 説明文 + 関連パターンへのリンク |
| パターン | 複数の事例で観察された再利用可能な型 | WikiXP 形式（下記） |
| 事例 | パターンを観察した実世界の具体例 | 概要 + 観察されたパターンへのリンク |

`wiki/index.md` が HomePage として全体のナビゲーションを提供する。

## ナレッジ追加のワークフロー

1. 新しい事例を `wiki/<slug>.md` として追加する
2. 「観察されたパターン」セクションでパターンページへリンクする
3. 既存パターンに合致する → パターンページの「既知の使用例」に事例を追記する
4. 新しいパターンを発見した → `wiki/<slug>.md` でパターンページを新規作成する
5. `wiki/index.md` の該当セクションにリンクを追加する

## パターンページの形式（WikiXP）

各パターンは WikiXP（Portland Pattern Repository）形式に準拠する：

- **タイトル** — パターン名（h1）
- **サマリー** — 1〜2 文でパターンを要約（h1 直後の地の文）
- **コンテキスト** — このパターンが適用される状況
- **問題** — 解くべき問題を **太字** で記述
- **フォース** — 問題を難しくする競合する制約・緊張関係（箇条書き）
- **解決策** — パターンの解決策を **太字** で記述し、図や表を続ける
- **結果コンテキスト** — 適用後の状況（得られるもの・失うもの）
- **既知の使用例** — 実世界での適用事例
- **関連パターン** — 他パターンへのリンク

## 分野について

分野（広告、コード、医療など）による分類は行わない。**パターン軸で整理する**ことを優先する。同じパターンが異なる分野に現れることがこのリポジトリの主要な発見になる。
